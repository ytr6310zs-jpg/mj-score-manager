# 目的
このファイルは、操作手順書・技術ドキュメントを統一したスタイルで記述するためのルールを示します。

# 基本ルール
- 1アクション1行、命令形で記述してください（例: 「ログインをクリックする」）。
- 手順には必ず「目的」「前提」「手順」「期待結果」「失敗時の対応」を含めること。
- 専有用語（プレイヤー、対局、Yakuman など）は用語定義に従う。

# 手順テンプレート（Markdown）
```
### 手順: タイトル
- 目的: <目的を1行で>
- 前提: <実行条件>
- 手順:
  1. <操作1>
  2. <操作2>
- 期待結果: <期待される表示/出力>
- 失敗時の対応: <対処手順>
```

# 画像（スクリーンショット）ルール
- 画像は `docs/images/` に配置すること。
- 命名規則: `step-<2桁>-<説明>.png`（例: `step-01-login.png`）。
- すべての画像に `alt` テキストとキャプションを付与すること。
- 必要であれば注釈（矢印、囲み、番号）を付ける。色だけで情報を示さないこと。

# ファイル／バージョン管理
- ドキュメントは Git 管理すること。編集は issue または task に紐づけ、`issue-<番号>/<短い説明>` ブランチで作業してください。
- コミットメッセージ例: `docs(user-manual): ログイン手順を追記 (#70)`
- 重要な変更は版履歴（`docs/manual.md` の版履歴表）に記載すること。

# PR と公開手順
- ドキュメント変更は PR を作成してレビュアー承認を得ること。
- CI でビルドが通ることを確認する（`npm run build`）。
- 画像追加時は必ず `docs/images/` に同時に追加して PR に含める。

# チェックリスト（レビュー用）
- [ ] 手順が命令形で1アクション1行に分かれている
- [ ] 期待結果が明記されている
- [ ] 画像に alt とキャプションがある
- [ ] 実機で手順を検証済み（検証者名と日付）
- [ ] 版履歴を更新した

# 参考テンプレート
- 例: ユーザーマニュアルは [docs/user-manual.md](docs/user-manual.md) を参照してください。
