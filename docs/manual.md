# 目的
この手順書は、本システムを利用するエンドユーザーおよび運用担当者が、日常的な操作（プレイヤー管理、対局スコア入力、統計参照、エクスポート等）を行うための手順を示します。

# 対象
- 対象者: サイト利用者（大会運営者、スコア入力者、閲覧者）
- 管理者: 管理者権限を持つユーザー向け操作は「管理者機能」節に記載します
- 前提条件: 有効なアカウント、インターネット接続、対応ブラウザ

# 用語定義
- 対局（マッチ）: 1 回のゲーム記録（日時、参加者、スコアなどを含む）
- プレイヤー: スコア管理対象の人物
- Yakuman: 役満マスタ（管理者が定義）

# 準備（利用前の確認）
- 対応ブラウザ: Chrome / Firefox / Edge（最新安定版を推奨）
- JavaScript と Cookie が有効であること
- 利用するアプリのURL（管理者より提供）
- 利用に必要なアカウント（メール／パスワード）と権限

# 操作手順の書き方（本書の記載ルール）
各手順は以下の形式で記載します。
- 目的: 手順の目的
- 前提: 実行条件（権限や前提データ）
- 手順: 番号付きで具体的操作を記載（1アクション1行）
- 期待結果: 操作後に画面や出力で確認すべき内容
- 失敗時の対応: よくあるエラーと簡易対処

以降の節はこの形式で利用者向けに具体手順を列挙します。

# トラブルシューティング（よくある障害）
- 症状: ログインできない
  - 原因候補: パスワード誤入力、アカウントロック、ネットワーク
  - 対処手順:
    1. パスワードを再入力またはパスワードリセットを試す
    2. 別ブラウザやプライベートモードで再試行
    3. 管理者に連絡（連絡時は時刻・操作内容・スクリーンショットを添付）

- 症状: 対局が保存できない／バリデーションエラー
  - 原因候補: 必須項目未入力、形式不正、セッション切れ
  - 対処手順:
    1. 必須項目が入力されているか確認
    2. ページをリロードして再度入力
    3. 継続的に発生する場合は管理者へログと操作手順を報告

# 主要操作（利用者向け：詳細）

## ログイン
- 目的: アプリにログインして利用を開始する
- 前提: 有効なアカウント（メールアドレス／パスワード）
- 手順:
  1. ブラウザでアプリのURL（管理者提供）にアクセスします
  2. 画面の「ログイン」ボタンをクリックするか、`/login` に移動します
  3. メールアドレスとパスワードを入力して「ログイン」をクリックします
  4. MFA が有効な場合は、表示される手順に従って認証コードを入力します
- 期待結果: ログイン後、ダッシュボード（対局一覧等）が表示されます
- 失敗時の対応: パスワードリセット、別ブラウザでの確認、管理者へ連絡

## プレイヤー登録（一般ユーザー／管理者）
- 目的: 新しいプレイヤーを登録する
- 前提: 管理者権限が必要な場合は管理者でログイン
- 手順:
  1. ナビゲーションで「Players（プレイヤー）」を開きます
  2. 「新規追加」または「Add player」ボタンをクリックします
  3. 名前、ニックネーム、初期ステータス等を入力します
  4. 「保存」をクリックして登録を完了します
- 期待結果: プレイヤー一覧に新しい行が追加されます
- 失敗時の対応: 入力必須項目を確認して再保存

## 対局（スコア）入力
- 目的: 新規対局を登録してスコアを保存する
- 前提: 登録するプレイヤーが事前に存在すること
- 手順:
  1. 「Matches（対局）」を開き、「新規対局」ボタンをクリックします
  2. 日時、参加プレイヤーを選択します
  3. 各プレイヤーのスコア（点数／順位）を入力します
  4. 必要に応じて Yakuman（役満）やメモを入力します
  5. 「保存」をクリックします
- 期待結果: 対局一覧に新しい対局が表示され、統計値が更新されます
- 編集/削除: 対局行の「編集」ボタンで修正、または「削除」→確認ダイアログで実行

## 統計・ランキングの参照
- 目的: 任意期間の集計結果・ランキングを確認する
- 手順:
  1. 「Stats（統計）」を開きます
  2. 期間フィルタ／プレイヤーフィルタを設定します
  3. 結果テーブルやグラフを確認します
- 期待結果: フィルタに応じた集計結果が表示されます

## CSVエクスポート
- 目的: 対局データをCSVでダウンロードする
- 手順:
  1. 対局一覧またはエクスポート画面で期間を指定します
  2. 「CSVエクスポート」ボタンをクリックします
  3. ダウンロードが開始されるのでファイルを保存します
- 出力例: 日時, プレイヤー名, スコア, 備考 等

## 管理者向け（限定機能）
- Yakuman（役満）管理: 管理画面 → Yakumans で追加・編集・削除
- バルクインポート／バックアップ: 大量データの登録や復元は管理者の専用手順に従うこと（詳細は運用担当へ要確認）

# 付録: 参考（上級者向け）
- ローカルで動作確認する最小手順（開発者向け）
```bash
git clone <repo-url>
cd mj-score-manager
npm install
# .env.local は管理者から配布される値を設定
npm run dev
```
- 重要なパスの例:
  - ソース: `app/`, `components/`, `lib/`
  - DB定義: `ddl/`, `migrations/`
  - 管理スクリプト: `scripts/`
