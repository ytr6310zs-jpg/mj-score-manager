# Copilot エージェント委任ガイド

このファイルは、Copilot エージェント（GitHub Copilot Chat）へ作業を委任するときの標準テンプレートと運用方針をまとめた参照ドキュメントです。

## 標準実行フロー

```
設計 → 実装 → テスト → 動作確認 → npm run build → コミット → push → PR作成
```

途中で失敗した場合は原因を解消してから次工程へ進み、未解消のままコミット・push しないこと。

---

## 包括委任テンプレート（コピー用）

作業開始時に以下をそのまま貼ると、エージェントが一連の作業を止まらず完遂します。

```
先に設計を行い、実装・テスト・動作確認・コミット・pushまで委任します。
軽微な設計判断は既存実装準拠で自動判断してください。
コード修正・関連テスト・必要最小限のドキュメント更新まで許可します。
npm run build は毎回実行し、可能なら npm run lint と npm test も実行してください。
コミットは自動で可、push も自動で可、PR作成も自動で可（ベースブランチ: develop）。
破壊的変更・機密情報関連・要件矛盾が発生した場合のみ確認して止まってください。
```

---

## 停止条件（エージェントが確認のために止まる場合）

エージェントは次の条件のみで停止します。それ以外は自律的に進めます。

| 条件 | 内容 |
|------|------|
| 要件矛盾 | 既存仕様と指示が明確に矛盾している |
| 破壊的変更 | データ移行・スキーマ変更・公開I/F変更が伴う |
| 高リスク変更 | 認証・認可・機密情報に関わる変更 |
| 環境不足 | 外部権限・環境起因で検証が不可能 |
| ブランチ粒度の競合 | 命名や粒度に複数の妥当案があり運用影響が大きい |

## 検証コマンド一覧

| コマンド | タイミング |
|----------|-----------|
| `npm run build` | コミット前（必須） |
| `npm run lint` | コミット前（可能なら必須） |
| `npm test` | コミット前（可能なら必須） |
| `npm run check:local-pages` | UI変更後（local Supabase 起動が必要） |

---

## PR本文に必須の記載項目

1. 設計概要（目的・対象・影響範囲）
2. 実装概要（変更内容の要約）
3. 実行した検証コマンドと結果
4. 手動または自動の動作確認内容
5. 既知の未解決事項
