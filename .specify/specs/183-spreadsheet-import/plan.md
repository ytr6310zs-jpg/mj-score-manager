---
title: PLAN - スプレッドシート入力による一括インポート
---

概要
--
Google スプレッドシートのテンプレートを起点に、対局入力をまとめて取り込む導線を追加する。テンプレートは 1 試合 1 列の主表と、役満専用テーブルの 2 段構成とし、Web アプリ側では入力内容をプレビューしてから行単位で確定する。

テンプレートの前提は、既存の Google Sheets 連携資産を流用できることを優先し、役満定義は `lib/yakumans.ts` を単一の正とする。役満コードの手入力は廃止し、役満名/コードのプルダウン入力へ統一する。プレーヤー照合は表示名一致を基本とし、曖昧一致候補を提示して誤登録を防ぐ。

PoC 範囲の固定
--
- 入力元は Google スプレッドシート URL のみ
- ファイルアップロード（CSV/XLSX）は PoC 範囲外
- 4 列方式（SCORE/YAKUMAN/YAKITORI/TOBI_TOBASHI）との互換入力は提供しない
- 重複レコードはスキップ運用（上書きなし）
- 権限は `admin` / `editor` のみ実行可能、`viewer` は不可
- 優先度は P1、期限は 2026-05-31

ステップ
--
1. テンプレートスキーマを確定する
   - 目的: 1試合1列の主表、役満テーブル（初期20行+可変追加）、設定シート連携を固定する
   - 変更箇所: 新規のテンプレート生成/定義資産、`lib/yakumans.ts` の参照方針
   - 完了条件: 40 試合までの主表入力ルール、役満テーブル構造、設定シートの参照ルールが定義されている
2. 取り込みパーサーと検証を実装する
   - 目的: 主表セル（score+flags）と役満テーブルを復元し、プレーヤー照合と妥当性検証を行う
   - 変更箇所: 新規 `lib/*` または `app/*-actions.ts`
   - 完了条件: 区切り/大小文字ゆれ吸収、`T/TB` 競合警告、役満重複の `count` 集約、曖昧一致候補の提示が可能
3. プレビュー付きインポート UI を実装する
   - 目的: インポート前に行単位で確認・除外し、警告項目を修正できる画面を提供する
   - 変更箇所: 新規インポートページ/フォーム、必要に応じてナビゲーション
   - 完了条件: モバイル幅でも確認・修正・確定操作ができる
4. 既存入力フローと併存させる
   - 目的: 現行の手入力フォームを残し、追加入力経路として提供する
   - 変更箇所: 対局入力ページへの導線、必要最小限の説明文
   - 完了条件: 既存フローの挙動を変えずに導線が追加される
5. テストと検証を行う
   - 目的: 解析ロジック、競合検知、行単位修正、モバイル表示の回帰を防ぐ
   - 変更箇所: `test/*` と必要な E2E
   - 完了条件: build / test / lint がすべて通り、手動確認でプレビュー修正が確認できる
6. PoC 指標を測定して判定する
   - 目的: 手入力方式に対する改善効果を定量評価する
   - 変更箇所: 検証記録（手動測定ログ）
   - 完了条件: 操作回数 30% 減、入力時間 20% 短縮の両方を中央値で満たす

検証
--
- ビルド: `npm run build`
- テスト: 解析ロジックの unit テスト、UI/E2E テスト
- Lint: `npm run lint`
- 手動確認: テンプレートの役満/プレーヤープルダウン、インポートプレビュー、`T/TB` 修正、行単位確定、モバイル幅表示
- PoC 測定: 同一データセット 3 回計測（中央値）で、操作回数/入力時間の改善率を算出

リスクとロールバック
--
- Google Sheets 連携に依存すると、共有設定や権限で取り込みに失敗する可能性がある
- 役満テンプレートの維持に失敗すると、プルダウン候補とアプリの正規定義がずれる可能性がある
- `T/TB` 競合を自動処理すると誤登録の可能性があるため、必ずプレビュー修正ステップを維持する
- 役満テーブルの固定行を超える運用時に入力漏れが発生する可能性があるため、行追加手順をテンプレートに明記する
- 重複判定キーの設計が弱いと、誤スキップ/二重登録の可能性がある
- ロールバックは、新規インポート導線を無効化し、既存の手入力フローだけを残す形で行う

実装追記（2026-05-11）
--
- `previewMatchImportAction` で `yakuman_types`（`is_active = true`）を取得し、役満の存在確認を DB 正に変更
- `parseSpreadsheetMatrix` には `yakuman_types` 由来の役満定義を渡して解決する
- `confirmMatchImportAction` で未登録プレイヤー/未登録役満を自動作成する
- プレイヤー未登録は非ブロッキング警告として扱い、プレビュー上で選択可能にする
- 未知役満は `役満名 / コード` と `コード:役満名` 形式を許容し、妥当なコードなら取り込み継続
- `import_dedupe_key` カラムとユニーク制約で重複インポートを防止
