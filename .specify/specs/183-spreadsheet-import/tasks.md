---
title: TASKS - スプレッドシート入力による一括インポート
---

- T1: テンプレート仕様を固定する — Done 条件: シート名、40 試合までの列構成、役満プルダウン、複数飛ばし/飛びの記法が spec/plan に明記されている
- T2: テンプレート生成方針を決める — Done 条件: Google スプレッドシートベースでコピー可能なテンプレート構成（主表 + 役満テーブル + 設定シート）と、共有/参照方法が明記されている
- T3: 取り込みデータ構造を定義する — Done 条件: 1 試合 1 列セルの `score[,flag...]` パース、プレーヤー照合、役満テーブル連携、焼き鳥/飛ばし/飛び復元ルールが決まっている
- T4: プレーヤー照合ルールを定義する — Done 条件: 表示名一致を基本とし、曖昧一致候補の提示方法が決まっている
- T5: 事前プレビュー UI を設計する — Done 条件: 行単位で取り込み/スキップを選べ、`T/TB` 競合をユーザーが修正できる画面仕様が決まっている
- T6: モバイル表示の確認観点を整理する — Done 条件: 幅の狭い画面で確認・確定操作ができる前提が決まっている
- T7: テスト観点を整理する — Done 条件: unit / E2E の対象と成功条件（build/test/lint すべて必須）が定義されている
- T8: 重複取り込みポリシーを定義する — Done 条件: 重複判定キーと重複時の挙動（スキップ）が定義されている
- T9: 権限制御を定義する — Done 条件: admin/editor/viewer の実行可否が定義されている
- T10: PoC 計測手順を定義する — Done 条件: 比較シナリオ、計測項目、サンプル数、合格ラインが定義されている
- T11: 役満テーブル運用を定義する — Done 条件: 初期 20 行 + 可変追加、`gameNo/player/yakuman/count`、同一役満重複は `count` 集約が定義されている
- T12: フォーマット互換方針を定義する — Done 条件: 旧 4 列方式を受け付けず新方式のみ対応することが定義されている
- T13: 優先度・期限を固定する — Done 条件: Priority=P1、期限=2026-05-31 が spec/plan に反映されている
- T14: 実装フェーズへ進む準備を完了する — Done 条件: spec / plan / tasks が揃い、ユーザー承認待ちの状態になっている
