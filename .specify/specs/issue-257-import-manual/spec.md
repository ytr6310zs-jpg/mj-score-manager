背景
--
Issue #257 では、スプレッドシート一括インポートの運用性を高めるため、UI 導線の追加、パーサー改善、ユーザーマニュアル修正を同時に実施する。現状はマニュアル遷移導線が弱く、一括インポート画面から運用用スプレッドシートを開く導線も不足している。

また、既存の区切り文字では現場入力の表記ゆれを十分に吸収できず、半角/全角スペースを含む入力が失敗する。既存挙動を壊さずに区切り文字を拡張し、回帰防止の自動テストを追加する必要がある。

目的
--
- ハンバーガーメニューに利用者マニュアル導線を追加する
- 一括インポート画面に環境別スプレッドシートを開く導線を追加する
- スコア/フラグの区切り文字に半角/全角スペースを追加する
- ユーザーマニュアルの該当セクションを移動・修正する
- 既存バリデーション/パーサーの互換性を維持しつつテストで担保する

ユーザーストーリー
--

### User Story 1 - マニュアルへ即時遷移できる (Priority: P1)
利用者はハンバーガーメニューから「マニュアル」を選択し、利用者マニュアルへ遷移できる。

**Independent Test**: ヘッダーのハンバーガーメニューに「マニュアル」項目が表示され、指定 URL が開くことを確認できる。

**Acceptance Scenarios**:
1. **Given** ログイン済み画面、**When** ハンバーガーメニューを開く、**Then** 「マニュアル」項目が表示される
2. **Given** メニュー表示中、**When** 「マニュアル」を押す、**Then** `https://ytr6310zs-jpg.github.io/mj-score-manager/user-manual.html` が開く

### User Story 2 - 一括インポート画面から運用シートを開ける (Priority: P1)
利用者は一括インポート画面のボタンから、実運用のスプレッドシートを新規タブで開ける。

**Independent Test**: 一括インポート画面に「スプレッドシートを開く」ボタンがあり、環境ごとに設定した URL を開くことを確認できる。

**Acceptance Scenarios**:
1. **Given** 一括インポート画面、**When** 画面を表示、**Then** 「スプレッドシートを開く」ボタンがプレビュー作成ボタンの左に表示される
2. **Given** local 環境、**When** ボタンを押す、**Then** `.env.local` で設定した URL が新規タブで開く
3. **Given** staging/production 環境、**When** ボタンを押す、**Then** Vercel environment で設定した URL が新規タブで開く

### User Story 3 - 区切り文字の表記ゆれを許容できる (Priority: P1)
利用者は半角/全角スペースを含む形式でも、既存形式と同様に一括インポートできる。

**Independent Test**: 既存区切り文字に加えて半角/全角スペースでも同等にパースでき、既存ケースが回帰していないことを自動テストで確認できる。

**Acceptance Scenarios**:
1. **Given** 既存の区切り文字を含む入力、**When** パースする、**Then** 従来どおり成功する
2. **Given** 半角スペース区切り入力、**When** パースする、**Then** スコア/フラグが正しく分解される
3. **Given** 全角スペース区切り入力、**When** パースする、**Then** スコア/フラグが正しく分解される

### User Story 4 - マニュアル記述が現行運用と一致する (Priority: P1)
利用者は修正済みマニュアルで、現行のテンプレート準備手順と区切り文字仕様を誤解なく確認できる。

**Independent Test**: マニュアルの該当セクション位置と記述内容が Issue 指示どおり更新されていることを確認できる。

**Acceptance Scenarios**:
1. **Given** マニュアル本文、**When** セクション構成を見る、**Then** 「スプレッドシート一括インポート」セクションが成績集計セクション直前へ移動している
2. **Given** テンプレート準備手順、**When** 手順を確認、**Then** 「テンプレートボタンをクリック」文言が削除され、同一スプレッドシート内コピー作成手順へ更新される
3. **Given** 主表記入ルール、**When** 区切り文字説明を確認、**Then** 半角/全角スペースが追加されている

受け入れ条件
--
- ハンバーガーメニューに「マニュアル」リンクを追加し、指定 URL へ遷移できる
- 一括インポート画面に「スプレッドシートを開く」ボタンを追加し、新規タブ遷移できる
- スプレッドシート URL は local では `.env.local`、staging/production では Vercel environment で管理する
- 既存区切り文字定義に半角スペース・全角スペースを追加する
- 既存の区切り文字入力はすべて従来どおり成功する
- バリデーション/パーサーの拡張に対して自動テストを追加し、回帰を防止する
- ユーザーマニュアルのセクション移動・文言修正・区切り文字追記が反映される

制約
--
- 既存挙動を壊さない最小差分で実装する
- local Supabase 前提の安全運用を維持し、ローカル開発で staging/production の URL/キーを混在させない
- 機密情報はコード/ドキュメント/ログに含めない
- CI/運用の最低条件として `npm run build` を通す。可能なら `npm test` と `npm run lint` も通す

影響範囲
--
- ヘッダー/ナビゲーション UI
- 一括インポート画面 UI
- 一括インポートのパース/バリデーション処理
- ユーザーマニュアル本文
- 関連ユニットテスト

テスト
--
- Unit: 区切り文字パース（既存 + 半角スペース + 全角スペース）、バリデーション回帰
- UI: ハンバーガーメニューのマニュアルリンク表示と遷移
- UI: 一括インポート画面の「スプレッドシートを開く」ボタン表示位置と新規タブ遷移
- Doc: マニュアルのセクション順・手順文言・区切り文字説明の更新確認
