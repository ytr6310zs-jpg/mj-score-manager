title: PLAN - Issue #257 一括インポート・マニュアル関連修正
---

概要
--
Issue #257 は UI 導線追加、環境別 URL 設定、パーサーの区切り文字拡張、ユーザーマニュアル修正を束ねた改善である。本計画では、既存処理を壊さないことを最優先に、既存定義への追加と回帰テストで安全に進める。

前提
--
- スプレッドシート URL は環境変数で注入する
- local は `.env.local`、staging/production は Vercel 環境変数を使用する
- 既存区切り文字定義に追加する方式で実装する
- ユーザーマニュアルは既存記載を移動・修正する

実装ステップ
--
1. 現状調査と変更点の確定
   - 目的: 変更対象コンポーネント/パーサー/マニュアル位置を確定する
   - 完了条件: 変更対象ファイルと既存区切り文字定義位置を特定できる
2. マニュアル導線をヘッダーへ追加
   - 目的: ハンバーガーメニューから利用者マニュアルへ遷移可能にする
   - 完了条件: メニュー項目追加と URL 遷移が機能する
3. 一括インポート画面に運用シート導線を追加
   - 目的: 「スプレッドシートを開く」ボタンを追加し環境別 URL を開く
   - 完了条件: ボタン位置が要件どおりで、新規タブ遷移できる
4. 環境変数の参照設計を反映
   - 目的: local/Vercel の運用に沿った URL 注入を確立する
   - 完了条件: `.env.local` と Vercel 環境で同一キー運用が可能になる
5. パーサー/バリデーションの区切り文字拡張
   - 目的: 半角/全角スペースを許容しつつ既存挙動を維持する
   - 完了条件: 既存ケース非回帰 + 新規ケース成功
6. 自動テストの追加・更新
   - 目的: 仕様追加と互換維持をテストで担保する
   - 完了条件: 半角/全角スペースケース、既存ケースの回帰テストが通る
7. ユーザーマニュアル修正
   - 目的: セクション移動、誤記修正、区切り文字追記を反映する
   - 完了条件: Issue 指示の 3 点が文書上で確認できる

検証計画
--
- 必須: `npm run build`
- 可能な限り実施: `npm test`、`npm run lint`
- 手動確認:
  - ハンバーガーメニューからマニュアル遷移
  - インポート画面ボタンの表示位置と新規タブ遷移
  - 代表入力（既存区切り/半角スペース/全角スペース）のプレビュー結果
  - マニュアルのセクション順と文言修正

リスクと対策
--
- リスク: 環境変数未設定でボタン遷移先が空になる
  - 対策: 未設定時の安全なガード（ボタン非活性/案内表示）を設計時に検討
- リスク: 区切り文字追加で既存パースに副作用が出る
  - 対策: 既存ケースの回帰テストを必須化
- リスク: マニュアル修正対象が別管理の場合、同PRで反映できない
  - 対策: 本体リポジトリで対象ファイルを確認し、別管理なら連携PRを明記

スコープ外
--
- 一括インポート機能の大規模仕様変更
- 新規フォーマット導入や入力方式そのものの再設計
- 認証/認可モデルの変更
