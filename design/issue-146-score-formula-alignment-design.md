# Issue #146 設計: 成績集計の計算方法を既存Excel資料に合わせる

- Issue: https://github.com/ytr6310zs-jpg/mj-score-manager/issues/146
- 添付資料: 2026.xlsx（Issue添付）
- 目的: 現行アプリの成績集計ロジックを、過去運用で使用してきたExcelの数式仕様へ一致させる

## 1. 要約

添付Excel（第1回〜第5回、通期成績）の主要数式を抽出した結果、計算は大きく以下に分類される。

1. 合計点: `SUM(...)`
2. 順位: `RANK(..., ..., 0)`（降順）
3. 着順率: `COUNTIF(..., n) / COUNT(...)`
4. 回避率/補集合: `1 - (...)`
5. 通期の比率: `A / B`

現行アプリも同系統の指標を持つが、順位処理や母数定義、同点時の扱いなどで差分が生じる余地がある。まずは「Excelを唯一の正」とする計算仕様レイヤーを明確化し、UI表示値をその仕様から生成する設計に寄せる。

## 1.1 確定事項（2026-04-09）

1. 順位ロジックは Excel 互換（`RANK` の競争順位）を採用する。
2. 率計算の分母は、3人打ち/4人打ちを合算した対局数を基本とする。
3. ラスト回数は、3人打ちでは3位、4人打ちでは4位をカウントする。

## 2. 抽出したExcel数式パターン（代表）

## 2.1 各回シート（第1回〜第5回）

- 合計点
  - 例: `=SUM(B4:E4)`, `=SUM(B4:K4)`, `=SUM(B4:W4)`
- 総合順位
  - 例: `=RANK(G4,$G$4:$G$7,0)`, `=RANK(M4,$M$4:$M$11,0)`
- 着順率（4人打ち例）
  - 例: `=COUNTIF(AJ4:BM4,1)/COUNT(AJ4:BM4)`（1着率）
  - 例: `=COUNTIF(AJ4:BM4,2)/COUNT(AJ4:BM4)`（2着率）
  - 例: `=COUNTIF(AJ4:BM4,3)/COUNT(AJ4:BM4)`（3着率）
  - 例: `=COUNTIF(AJ4:BM4,4)/COUNT(AJ4:BM4)`（4着率）
- 個別指標の順位
  - 例: `=RANK(B4,B4:B12,0)`
  - 例: `=RANK(AG4,AG4:AG15,0)`

## 2.2 通期成績シート

- 合計: `=SUM(B3:FY3)`
- 比率: `=GD3/GB3`, `=GE3/GB3` など
- 補集合: `=1-GH3`, `=1-GG3-GH3`

## 2.3 備考

- `RANK` は同値同順位（Excelの競争順位）で、次順位はスキップされる。
- 一部に `#REF!` を含む数式パターンが存在するため、移植対象は「有効な式が定義しているルール」を優先する。

## 3. 現行実装との差分候補

対象実装:
- [lib/stats.ts](lib/stats.ts)
- [lib/metric-ranks.js](lib/metric-ranks.js)
- [app/stats/page.tsx](app/stats/page.tsx)

主な差分候補:

1. 総合順位付けの定義
- 現行: 合計点でソート後に配列インデックスで順位付け（同点時に連番）
- Excel: `RANK` 相当（同点は同順位、欠番あり）
- 影響: 表示順位、順位バッジ、同点時の見え方

2. 母数（分母）定義の明示
- 現行: 多くは `games`（対局数）を分母にしている
- Excel: `COUNT(着順セル範囲)` を分母にしている
- 影響: 欠損値や対象外局が含まれる場合の率

3. 指標の公式化（単一ソース化）
- 現行: ロジックが集計処理に散在
- 目標: 「Excel準拠の計算仕様」を関数として集約
- 影響: 回帰防止と将来保守性

## 4. 設計方針

## 4.1 仕様レイヤーを分離

新規モジュール（案）を追加し、計算式を関数として宣言的に定義する。

- 追加案: `lib/stats-formulas.ts`
- 役割:
  - 合計
  - 順位（Excel互換 `RANK`）
  - 着順率（`COUNTIF / COUNT` 相当）
  - 回避率（`1 - x`）

これにより、`fetchPlayerStats` は「データ収集」と「計算適用」を分離できる。

## 4.2 順位アルゴリズムをExcel互換へ

Excel `RANK`（降順）に合わせる。

- 定義（競争順位）:
  - 値 `v` の順位 = `1 + (v より大きい値の個数)`
- 例: 値 `[100, 100, 90]` の順位は `[1, 1, 3]`

この方式を以下に適用する。

- 総合順位（現在 `PlayerStats.rank`）
- 必要に応じてメトリクス別順位表示

## 4.3 4人打ち/3人打ち混在時の着順率

本Issueの確定ルール:

- 分母は 3人打ち/4人打ちの合算対局数（`games`）を使用する。
- ラスト回数は `rank === playerCount` で判定する。
  - 3人打ち: `rank === 3`
  - 4人打ち: `rank === 4`
- ラス回避率は `1 - lastCount / games` とする。

備考:
- 将来 4着率を新規指標として追加する場合は、別仕様として分母定義を明示する。

## 4.4 表示レイヤーへの反映

- 指標名と式の対応をドキュメント化し、実装と乖離しないよう管理

## 7. リスクと確認事項

1. 添付Excel内に `#REF!` を含む式があるため、移植対象式の確定が必要
2. 通期シートの列意味（GB, GD, GE...）は列見出し対応を確認しないと誤実装の恐れ
3. 同点順位の扱い変更により、既存表示順やユーザー認知が変わる可能性

## 8. 実装条件の確定内容（2026-04-09 更新）

ユーザー回答により、以下を確定した。

1. 通期シート列の意味
- 左から `対局数`, `トップ回数`, `ラス回数`, `飛ばし回数`, `飛び回数`, `焼き鳥回数`

2. 補集合系指標の意味
- 左から `ラス回避率`, `飛び回避率`, `焼き鳥回避率`, `接待率`

3. `#REF!` の背景
- `#REF!` は「試合未実施列で順位式の参照先が無効」なために発生
- 試合実施列の順位式は現役で有効指標として扱う

4. 今回の反映範囲
- [app/stats/page.tsx](app/stats/page.tsx) の既存成績指標のみを対象とする
- 通期シート上にのみ存在し、画面非表示の指標は今回対象外
