## コンテキスト

現在のヘッダ（`components/app-header.tsx`）はメニュー周りが縦に並ぶため、ダッシュボード上部で縦方向の表示領域を多く消費しています。主要な操作は3つ（スコア入力 / 対局履歴 / 成績集計）で、これらを横並びに配置することでページ上部の縦スペースを節約したい。

## 問題

- 小さい画面でメニューが縦に並び、上部の UI スペースが圧迫される
- 既存のボタンは `w-full sm:w-auto` のため小画面でフル幅になりやすい

## 決定

1. 主要メニューは横並び（横方向に並べる）をデフォルトの動作とする。
2. 非常に狭い画面では折返し（wrap）やアイコン化／ハンバーガー化で対応する（実装は段階的に検討）。
3. 実装はこのブランチ（`feature/issue-96`）で行い、PR を通して段階的にレビューする。

## 実装方針（詳細）

1. `components/app-header.tsx` のメニューコンテナを以下のように修正する。
   - 変更前: `grid grid-cols-2 gap-2 sm:flex sm:flex-wrap sm:items-center`
   - 変更後案: `flex items-center gap-2 flex-wrap`

2. 各メニューのボタンから `w-full sm:w-auto` を削除し、`w-auto` を使うことで小画面でもフル幅にならないようにする。

3. 必要に応じて次の拡張を検討する：
   - ラベルを省略してアイコンのみ表示（極小画面）
   - ハンバーガーメニューで折りたたむ（アニメーション、アクセシビリティ対応）

## 受け入れ基準

- 主要3メニューが横並びになる（幅が十分な画面で1行に並ぶ）。
- モバイル（例: 360px 幅）で縦に不必要な高さを取らない。
- キーボード/スクリーンリーダーでの操作性が維持される。
- 既存のリンク動作（現在ページ判定での default/outline 切替）が維持される。

## 実施手順

1. `AppHeader` を解析し影響範囲を特定（ボタンの配置、className、他のヘッダ要素）。
2. クラス修正を行い、ローカルでビルド・表示確認。
3. モバイル幅での視覚検証（`/matches`, `/stats` 等）。
4. PR を作成、スクリーンショットを添えてレビュー依頼。

## リスク

- 既存のスタイルやレスポンシブ動作を壊す可能性があるため、段階的に変更し、スクリーンショットで差分確認する。
