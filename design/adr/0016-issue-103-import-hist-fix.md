# ADR 0016: 過去対戦インポート修正 (Issue #103)

日付: 2026-04-08

## 概要

過去対戦のインポート処理に関する設計決定を記録する。Issue #103 の指摘に基づき、複数回のインポートを想定しないため、`備考`（`notes`）フィールドを用いて取り込み情報を管理する仕組みを廃止する。

## 背景

現状、歴史的な対局データを CSV 等から取り込むスクリプト（`scripts/import-historical-games.mjs` 等）が、取り込み元などのメタ情報を対局の `notes`（備考）フィールドに追記している箇所がある。これは取り込みの重複検出や参照のために便利だが、運用上「複数回インポートは想定しない」ため、`notes` を機械的な管理用途へ使う必要はないという判断が Issue #103 で示された。

## 問題点

- `notes` に取り込みメタを埋め込むと、ユーザーが自由に編集する `notes` と運用データが混在し、将来の混乱や誤検出の原因となる。
- `notes` に依存する実装があると、既存データに対する後続修正やリストア時に不整合が発生しやすい。

## 決定

1. インポート処理で対局の `notes` フィールドに取り込みメタ（例: "imported-from: ..."）を書き込まない。既存の `notes` 書き込みロジックを削除または無効化する。
2. 既に `notes` に書き込まれたメタ情報は即時に削除する必要はない（履歴保持）。ただし、必要ならクリーンアップ用スクリプトを別途用意する。
3. 将来的に取り込みの管理が必要になった場合は、専用の列（例: `import_source`）または専用テーブル（`imports`）を設ける設計を採用する。現時点では対応不要のため保留とする。

## 実装方針

1. `scripts/import-historical-games.mjs` の修正
   - `notes` に追記している箇所を削除する、あるいは追記をスキップするオプションをデフォルト有効にする。
2. `lib/matches.ts` の確認
   - `fetchMatchResults` の実装が `notes` の特定パターンを解釈している箇所があれば削除する。
3. テスト
   - インポートスクリプトのユニット/統合テストを追加または更新し、`notes` に書き込まれないことを検証する。
4. ドキュメント
   - `README.md` または `scripts/README` にインポートの挙動変更（`notes` を用いない）を記載する。
5. オプション: 既存データのクリーンアップ
   - 必要に応じて、`notes` から自動で付与した取り込みタグを除去するスクリプトを追加する（別 PR）。

## テストケース

- 既存の CSV を import しても `games.notes` に新しい import タグが追加されないこと。
- 既存の `notes` 内容は改変されないこと。

## リスクと緩和

- リスク: 既存運用が `notes` を参照していた場合、参照元が無効化される。\
  緩和: 変更点を PR とドキュメントで明示し、必要なら移行手順を提示する。
