## コンテキスト

Issue #140 では、ヘッダーメニュー押下後に表示される現在のローディング表現が「固まって見える」課題を改善することが求められている。

現状は App Router の `app/loading.tsx` が挿入されるため、ページ切替直前に画面上部だけが一瞬変化し、ユーザーには「画面が切替途中」より「表示が止まった」印象を与えやすい。

## 現状整理

- `app/loading.tsx`
  - 画面最上部に細いバーを表示し、`animate-pulse` で点滅。
  - `aria-live="polite"` で読み込み中を通知。
- `components/app-header.tsx`
  - `/`, `/matches`, `/stats` の遷移リンクで pending 表示を実装済み。
  - `usePathname` 変化で pending を解除。
  - 見た目は `opacity-70 cursor-wait` のみで、遷移中の意味が弱い。

## 問題

- `loading.tsx` はページ遷移中にレイアウト差し込みが起き、視覚的に不自然。
- 点滅バーは意味が伝わりにくく、静止して見えるケースがある。
- pending のフィードバックがリンク自身の薄い変化だけで、操作が受理された実感が不足。

## 決定

1. グローバル fallback である `app/loading.tsx` は撤去する。
2. 画面遷移中の主フィードバックは `AppHeader` のインライン表示に一本化する。
3. 遷移開始直後に、押下リンクの状態変化 + ヘッダー内の「遷移中」インジケーターを同時表示する。
4. UI 層のみ変更し、API/DB/認証ロジックは変更しない。

## 実装方針

### 1) グローバルローディングの撤去

- 対象: `app/loading.tsx`
- 対応: ファイルを削除。
- 目的: ルート切替時の差し込み表示をなくし、固まって見える印象を低減する。

### 2) ヘッダーの遷移中表示を強化

- 対象: `components/app-header.tsx`
- 変更内容:
  - pending 対象に `admin` を追加する（メニュー遷移も同品質に揃える）。
  - 遷移中リンクは以下を適用。
    - `pointer-events-none`（連打防止）
    - `cursor-wait`
    - 右端に小型スピナー（CSS 回転）
  - ヘッダー右上またはリンク群下に「遷移中...」バッジを追加。
    - 例: `inline-flex items-center gap-1 rounded-full border bg-card/90 px-2 py-1 text-xs`
    - 左にドットまたは小リングを回転表示して、静止に見えない動きを明示。
  - `aria-live="polite"` で「画面遷移中」を読み上げ通知。

### 3) 表示制御

- pending state:
  - 開始: リンク onClick
  - 終了: `usePathname` 変化時
- 表示安定化（任意だが推奨）:
  - 120ms 以上遷移が続く場合のみ「遷移中」バッジを出す。
  - 目的: 高速遷移時のチラつき抑制。

## 受け入れ基準

- ヘッダーリンク押下直後に、押下リンクが即座に遷移中表示へ切り替わる。
- 遷移待機が長い場合、ヘッダー内「遷移中」表示が明確に視認できる。
- `/admin` への遷移でも同様のフィードバックが出る。
- 画面遷移時に `loading.tsx` 由来の上部バー差し込みが発生しない。
- キーボード操作・スクリーンリーダー利用時に遷移中が把握できる。

## 影響範囲

- 変更: `components/app-header.tsx`
- 削除: `app/loading.tsx`
- 非変更: DB, API, 認証ロジック, サーバーアクション

## リスクと対策

- リスク: pending が解除されないケース（想定外の遷移失敗）があると UI が待機状態に残る。
  - 対策: 保険としてタイムアウト解除（例: 10 秒）を実装候補に含める。
- リスク: スピナー追加で視覚ノイズが増える。
  - 対策: サイズを最小化し、現在 active タブの視認性を崩さない配色にする。

## テスト観点

1. `/`, `/matches`, `/stats`, `/admin` への遷移で pending 表示が出る。
2. 連続クリックしても同一リンクが連打されない。
3. 遷移完了時に pending 表示が消える。
4. 画面上部バー（旧 loading.tsx）が表示されない。
5. `npm run build` が成功し、型エラーがない。

## 実施ステップ

1. `components/app-header.tsx` の pending 型と UI を拡張。
2. `app/loading.tsx` を削除。
3. 手動確認（4導線）。
4. `npm run build` 実行で回帰確認。
