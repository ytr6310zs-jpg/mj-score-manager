## 概要

成績集計画面で 1位 と 3位 の見た目が近く、一覧を見たときに順位差を素早く認識しづらい。
3位の表示を中心に配色を見直し、順位バッジ・行背景・指標ハイライトの視認性を上げる。

Issue: https://github.com/ytr6310zs-jpg/mj-score-manager/issues/135

## 背景 / 問題

現在の成績集計 UI では以下の配色を使っている。

- 1位: yellow 系
- 2位: slate 系
- 3位: amber / orange 系

1位 と 3位 がどちらも暖色系で、特に次の箇所でパッと見の差が弱い。

- モバイルカードの順位バッジ
- モバイルカードの行背景
- PC テーブルの順位バッジ
- 各指標セルの 1/2/3 位ハイライト

さらに順位配色が複数ファイルに重複定義されており、個別修正すると表示差分が発生しやすい。

- `app/stats/page.tsx`
- `components/stats-sortable-table.tsx`

## 決定

1. 3位の色を「1位の金色に近い明るい暖色」から外し、より暗く深い銅色系に変更する。
2. 順位表示に使うスタイル定義を共通化し、モバイルカード・PC テーブル・指標ハイライトで同一のトークンを使う。
3. 1位 / 2位 / 3位 の意味付けは維持しつつ、3位のみコントラストを上げる最小差分で対応する。
4. issue #119 で導入した「各指標の 1〜3 位ハイライト」にも同じ配色を適用し、順位表現を統一する。

## デザイン方針

### 1) 色の役割

- 1位: もっとも明るく目立つ金系
- 2位: 中立的な銀 / slate 系
- 3位: 金系と離れた、暗めの銅 / orange-brown 系

3位は「ブロンズらしさ」は残しつつ、1位と並んだときに別グループとして認識できる色にする。

### 2) 変更対象

- 順位バッジ
- モバイルカードの順位行背景
- テーブル中の順位バッジ
- 指標セルの上位 1〜3 位バッジ

### 3) 実装上の共通化

共通の順位スタイル定義を追加する。候補:

- `lib/rank-theme.ts`
- または `lib/stats-rank-theme.ts`

このモジュールで少なくとも以下を管理する。

- `RANK_BADGE_STYLES`
- `RANK_ROW_BG_STYLES`
- 必要なら `RANK_LABELS`

これにより `app/stats/page.tsx` と `components/stats-sortable-table.tsx` の重複定義を解消する。

## 具体案

### 配色案

最終調整は実画面確認で行うが、方向性は以下とする。

- 1位: 現状の yellow 系を維持
- 2位: 現状の slate 系を維持
- 3位: 現状の `bg-amber-400/70 text-amber-900` から、より深い orange / brown 系へ変更

例:

- badge: `bg-orange-700 text-orange-50`
- row background: `bg-orange-100/80`

目的は「3位を暗くすること」ではなく、「1位と混同しにくくすること」である。

## 受け入れ基準

- 成績集計画面で 1位 と 3位 を一覧で見比べたとき、即座に区別できること。
- モバイルカードと PC テーブルで順位配色が一致していること。
- 各指標セルの 3位ハイライトも同じ配色ルールに従うこと。
- 順位配色の定義が 1 か所に集約され、今後の微調整を単一修正で行えること。

## 影響範囲

- `app/stats/page.tsx`
- `components/stats-sortable-table.tsx`
- 共通スタイル定義の新規ファイル

## 実装ステップ

1. 共通の順位スタイル定義を新規作成する。
2. `app/stats/page.tsx` の `RANK_ROW_BG` と `RANK_BADGE` を共通定義参照へ置き換える。
3. `components/stats-sortable-table.tsx` の `RANK_BADGE` を共通定義参照へ置き換える。
4. 3位の配色を調整し、モバイルカード・テーブル・指標ハイライトで反映を確認する。
5. `npm run build` を実行して型・ビルドに問題がないことを確認する。

## 代替案

1. 3位のテキストやアイコンを増やして区別する。
   - 色変更より影響が大きく、UI の密度も上がるため今回は採用しない。
2. 1位側の色を変更する。
   - 1位は現状でも直感的で、今回の課題は主に 3位側にあるため優先しない。
3. カード背景のみ変更する。
   - テーブルや指標ハイライトとの不整合が残るため採用しない。

## メモ

issue #119 の実装で順位色が視覚言語として広がったため、今回の修正は単なる色替えではなく「順位表示トークンの統一」として扱う。
