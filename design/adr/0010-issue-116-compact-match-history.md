## コンテキスト

Issue #116（対局履歴 省スペース化）では、次の3点が要求されている。

1. 一位の王冠アイコンは不要
2. 飛び／飛ばし等の情報が重複しているため、どちらかに寄せる
3. 合計 0 の表示は不要

現状の対局履歴UI（`app/matches/page.tsx`）では、以下が確認できる。

- `resultBadge()` が 1位に `🏆` を表示している
- 飛び/飛ばしは「プレイヤー行バッジ」と「対局単位の要約（飛び: X / 飛ばし: Y）」の両方で表示される
- 合計スコア（`match.scoreTotal`）がモバイル/PCの一覧UIに表示されている

## 問題

- 情報密度が高く、特にモバイル表示で縦方向の占有が大きい
- 同一情報が複数箇所に表示され、視線移動コストが増える
- ユーザー判断に不要な合計スコア表示がノイズになっている

## 決定

1. 1位の王冠表示を廃止する。
   - `resultBadge()` での `rank === 1` 分岐を削除する。
   - 順位情報は既存の `1位/2位/...` ラベルで十分とする。

2. 飛び/飛ばし/焼き鳥の表示は「プレイヤー行バッジ」に集約する。
   - プレイヤー行のバッジ表示（飛び・飛ばし・焼き鳥）は維持する。
   - 対局単位の集約表示（モバイルの `dl` ブロック、およびPC表の「飛び/飛ばし」列と「焼き鳥」列）は削除する。

3. 合計スコア表示をモバイル/PCともに廃止する。
   - モバイルの見出しから `合計 ...` 表示を削除する。
   - PC表の「合計」列自体を削除する。

## 実装方針（詳細）

1. バッジ判定ロジックの整理
   - `resultBadge()` から王冠判定を除去する。
   - 既存の飛び・飛ばし・焼き鳥の優先順位は維持する。

2. モバイルカードの省スペース化
   - `dl` の「飛び/飛ばし」「焼き鳥」項目を削除する。
   - 備考は情報価値が高いため維持する。

3. PCテーブルの省スペース化
   - 「飛び/飛ばし」「焼き鳥」列を削除する。
   - 列削除に合わせてヘッダー・ボディのカラム構成を再調整する。

4. 合計表示の削除
   - `match.scoreTotal` は内部整合性や将来利用のためデータとしては保持する。
   - 一覧画面では参照しない構成に変更する。

## 受け入れ基準

- 1位プレイヤーに王冠アイコンが表示されない。
- 飛び/飛ばし/焼き鳥情報が、対局行内で重複表示されない。
- 合計スコアがモバイル/PCの一覧UIに表示されない。
- モバイル/PCの両表示でレイアウト崩れがない。

## 影響範囲

- 変更: `app/matches/page.tsx`
- 非変更: DB スキーマ、API、保存データ形式、認証

## 代替案

1. 対局単位の要約のみ残し、プレイヤー行バッジを削除する。
   - 一覧性は上がるが、「誰が飛び/飛ばしだったか」の対応関係が弱くなるため不採用。

2. 合計スコアを 0 のときだけ非表示にする。
   - 条件付き表示は可能だが、ルールがUI上で分かりづらく、空欄が欠損値に見えるため不採用。

## テスト / 検証

- 手動確認
  - モバイル表示でカードの縦サイズが縮小していること
  - PC表示で列削減後も可読性を維持できていること
   - 合計表示を削除しても一覧の可読性が維持されること
- 自動確認
  - `npm run build` を実行し、型・ビルドエラーがないことを確認する

## リスク

- バッジ集約により、対局単位での情報俯瞰がしづらくなる可能性がある。
- 合計表示を削除することで、内部上は存在する値を画面確認できなくなる。
- 将来、運用上の確認需要が出た場合は詳細画面やCSVで補完する。

## 実施ステップ

1. `app/matches/page.tsx` の表示ロジックをADRどおりに変更。
2. `npm run build` を実行して品質確認。
3. 主要表示（モバイル/PC）を手動確認。
