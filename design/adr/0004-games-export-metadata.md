# ADR 0004: `games` エクスポートにおける拡張項目の取り扱い

日付: 2026-03-28
作成者: プロジェクト（個人開発）

## 背景
現行の `games` テーブルにはすべての将来的に必要となる項目が存在しない場合がある。CSV エクスポート時の拡張性と互換性を両立させる必要がある。

## 決定
- コアカラム（既存の `games` カラム）をそのまま CSV の固定ヘッダとして出力する。
- 拡張項目は JSON 形式でまとめて `metadata_json` カラム（文字列）として出力する。CSV ヘッダには `metadata_json` と `export_version` を追加する。
- `metadata_json` は key/value のフラットなオブジェクトを想定（必要に応じてネスト許容）。インポート時は JSON をパースして利用する。

## 理由
- スキーマ変更を最小限に抑えつつ、新しいフィールド追加を容易にするため。
- 既存ツールはコアカラムのみ参照すれば互換性を保持できる。
- 解析や詳細表示が必要なツールは `metadata_json` をパースして利用できる。

## CSV ヘッダ（例）
```
<core_columns...>,metadata_json,export_version
```
- `export_version` は今後の互換性対応のために必須とする（例: v1）。

## インポート方針
- インポート処理はまず `export_version` を確認する。
- `metadata_json` の内容はバリデーション（必要に応じてスキーマ）を行い、既知フィールドは個別カラムにマップできるようにする。

## 影響・運用
- DB 側は `metadata_json` をオプショナルなビューに持たせる（NULL 許容）。
- 既存のエクスポート処理はコアカラムに `metadata_json` を追加する形で移行する。
- ADR を `design/adr/` に保存し、変更は PR でレビュー・マージする。

*** End ADR
